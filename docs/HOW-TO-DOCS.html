Setting up Sphinx

The docs for BoolForge are made using the Python package Sphinx. Make sure you have Sphinx installed, which can be done via pip install -U sphinx, or some other manner as described in the Sphinx installation guide.

Additionally, BoolForge uses the third-party theme known as Read the Docs (rtd). Third-party themes require an additional installation which can be done via pip install sphinx-rtd-theme.

Building the HTML

Once Sphinx is set up, the docs can be built by running the build script make.bat and passing the html argument. This can be done by navigating to the main folder and running .\docs\make.bat html. On MacOS, you may not need to include the file extension. It may display a few errors. If the file returns build succeeded, then everything is great.

Hosting the Documentation

The documentation is hosted on the /pages/ branch of the repository, but we are currently working in the /main/ branch. As such, each build will need to be manually ported over to the other branch, specifically the contents of the /docs/_build/html/ folder.
